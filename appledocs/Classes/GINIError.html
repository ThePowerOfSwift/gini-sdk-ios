GINIError Class Reference

Inherits from NSError
Declared in GINIError.h

Overview

The common error class to make errors which are from Gini (and which are somehow expected to happen) distinguishable from other errors (e.g. network errors, argument errors, …). All errors from Gini are an instance of this class or an instance of a subclass of this class.

Of course we could use “normal” NSError instances and make them distinguishable by the error domain, but we think it is better to use the type system, especially since Objective-C has a nice type system with subclasses, since it is always possible to upcast to NSError.
